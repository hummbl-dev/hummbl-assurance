# Security Policy

## Supported Versions

Only the latest code on the default branch is supported for security fixes unless a tagged release line is explicitly documented otherwise.

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities.

Report vulnerabilities privately to:
- security@hummbl.io
- reuben@hummbl.io

Include:
- a clear description of the issue
- affected files, tags, commits, or URLs
- reproduction steps or proof of concept
- expected impact and likely attack path

We aim to acknowledge reports within 72 hours and coordinate remediation and disclosure after validation.

## Scope

This policy covers:
- receipt verification and assurance logic
- compatibility and revalidation logic
- conformance schemas and fixtures
- release and protection audit workflows

## Disclosure

HUMMBL follows coordinated disclosure. Please allow time for validation, patching, and release before public disclosure.
